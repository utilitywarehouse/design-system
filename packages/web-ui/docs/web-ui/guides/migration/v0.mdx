# v0

It is possible to use Web UI in an application which is currently using
`customer-ui-material`. While you probably don't want to use components from both
libraries side-by-side,

this is no longer true - make it apparent that this only applies to certain components - as long as any Web UI are children of a `ThemeProvider`
from Web UI then you can do this at the most specific level you want, whether
that is app-wide, for a single page, or for a single section or component.

## Theme

We advise against using the `theme` object, and it is highly likely it will be removed in a future release.

As we can't reliably predict the theme usage there is no codemod for updating
any theme values, so you'll have to update manually.

the components key will have changed considerably, you shouldn't rely on it.

### Breakpoints

```diff
{
 "breakpoints": {
-  "keys": ["mobile", "tablet", "desktop"],
+  "keys": ["mobile", "tablet", "desktop", "wide"],
   "values": {
     "mobile": 0,
-    "tablet": 768,
-    "desktop": 1366
+    "tablet": 740,
+    "desktop": 992,
+    "wide": 1200
```

Use the mobile first mediaQueries instead of the theme.breakpoints helper fucntions...
doesn't rely on ThemeProvider

````diff
+ import { mediaQueries } from '@utilitywarehouse/web-ui';

[...]

 sx={{
   color: 'red',
-  [theme.breakpoints.up('desktop')]: {
-    color: 'blue',
-  },
+  [mediaQueries.desktop]: {
+    color: 'purple',
+  },
}}
```
````
